# Security Policy

## Reporting a Vulnerability

Use this section to tell people how to report a vulnerability.

Tell them where to go, how often they can expect to get an update on a
reported vulnerability, what to expect if the vulnerability is accepted or
declined, etc.

eg:
5323/path/to/file.py
lines: 130 - 146
[error message]
specific issue (varable, constant etc)
version (if using an older version this issue may be resolved in a newer version)
importance: 
1=minor 
2=look into
3=worsens game experience 
4=put focus onto immediately
5=major security/game breaking issue
